# Syntax Guide

This document defines the syntax conventions for documentation and manuscript
content inside the `template_methods_paper` exemplar.

---

## 1. Markdown Links

Hyperlinks must be informative. Never use placeholder text.

- **BAD**: [this link](https://github.com/docxology/template) to see the template.
- **GOOD**: See the [Research Project Template](https://github.com/docxology/template).

For internal cross-references inside `manuscript/`, prefer relative paths.

---

## 2. Pandoc-Crossref Cross-References

Inside `manuscript/` files, use Pandoc-crossref `[@label]` syntax. **Never**
use raw LaTeX `\ref{}` macros — they render literally in HTML/slide outputs
and bypass auto-numbering. Never hardcode figure or table numbers.

- **BAD**: See Table 1 or Figure 2.
- **GOOD**: See [@tbl:compiled_plans] or [@fig:step_counts].

### Figure Label Registry

The one figure the analysis script produces:

| Anchor (in `03_results.md`) | PNG Filename | Generator |
|---|---|---|
| `{#fig:step_counts}` | `output/figures/step_counts.png` | `scripts/methods_analysis.py` (matplotlib bar chart) |

The table anchor used in `03_results.md`: `{#tbl:compiled_plans}`.

---

## 3. Numbers in Prose (`{{TOKEN}}` injection)

This exemplar **uses** `{{VARIABLE}}` token injection — the full table lives
in [`../manuscript/SYNTAX.md`](../manuscript/SYNTAX.md). Every numeric or
run-dependent value in `manuscript/*.md` must be a `{{TOKEN}}` generated by
`src/manuscript_variables.py::generate_variables`, never a hand-transcribed
literal.

- **Prefer**: `{{PBS_STEP_COUNT}}`, `{{DETERMINISM_CHECK}}`, etc.
- **Avoid**: writing the literal generic placeholder text `{{TOKEN}}` as
  illustrative prose inside a rendered manuscript section — the renderer
  treats any `{{UPPERCASE_NAME}}` pattern as a real token to resolve and
  fails the build if it cannot. Describe the mechanism without double curly
  braces instead (e.g. "a generated value", "a resolved variable").

If you state a fixed number that is a static code constant rather than a
per-run-generated value (e.g. the genesis hash length), add a row to
`data/claim_ledger.yaml` recording its source instead of inventing a token.

---

## 4. Code Blocks

Always tag code blocks with their language identifier for Pandoc syntax
highlighting.

```python
def example() -> bool:
    return True
```

```bash
uv run pytest projects/templates/template_methods_paper/tests -v
```

For inline code referencing file paths, use single backticks:
`projects/templates/template_methods_paper/src/methods_dsl/compiler.py`.

---

## 5. Table Captions (Pandoc)

Place the caption below the table; the `{#tbl:label}` goes on the caption
line:

```markdown
| Method | Steps | Target |
|---|---|---|
| PBSPreparation | {{PBS_STEP_COUNT}} | {{PBS_TARGET}} |

: Compiled-plan summary written by the analysis script. {#tbl:compiled_plans}
```

Do not use a `Table:` prefix and do not hardcode the table number — use
`[@tbl:compiled_plans]`.

---

## 6. Adding a New Figure or Token

**Figure:**

1. Add the plotting call to `scripts/methods_analysis.py` and write the PNG
   to `output/figures/`.
2. Add the Pandoc image reference in `03_results.md`:
   ```markdown
   ![Caption text describing the figure.](../output/figures/my_figure.png){#fig:my_label}
   ```
3. Reference it: `See [@fig:my_label].`
4. Update `manuscript/AGENTS.md` and `manuscript/SYNTAX.md`'s registry, then
   re-run the pipeline.

**Token:**

1. Add the key to `generate_variables()` in `src/manuscript_variables.py`.
2. Add a test in `tests/test_manuscript_variables.py`.
3. Reference `{{YOUR_TOKEN}}` in a manuscript section.
4. Update `manuscript/SYNTAX.md`'s token table.

---

## 7. LaTeX Math in Manuscript

Use `$...$` for inline math and `$$...$$` for display equations. For
numbered, referable equations, attach a Pandoc-crossref anchor with
`{#eq:label}` after the closing `$$` and reference with `[@eq:label]`. (This
exemplar's prose is largely equation-free; the convention is documented for
forks that need it.)
