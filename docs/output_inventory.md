by CI artifact upload.

## Adding a New Output File

1. Compute the new value in a tested `src/methods_dsl/` function (or
   `src/manuscript_variables.py` for a token).
2. Write it from `scripts/methods_analysis.py` (or
   `scripts/z_generate_manuscript_variables.py` for a token) under the
   correct `output/` subdirectory.
3. Append a row to the table above naming the file, role, generator, and
   stage.
4. If the manuscript references it, add a `{{TOKEN}}` or `{#fig:label}` in
   the appropriate `manuscript/*.md` section.
5. Re-run the analysis and confirm the new file appears.

## Regeneration Sequence

```bash
# 1. Clean (optional)
rm -rf projects/templates/template_methods_paper/output/

# 2. Run analysis (produces data/, reports/, figures/)
uv run python projects/templates/template_methods_paper/scripts/methods_analysis.py

# 3. Generate + inject manuscript variables
uv run python projects/templates/template_methods_paper/scripts/z_generate_manuscript_variables.py

# 4. Render PDF (produces pdf/, web/)
uv run python scripts/pipeline/stage_03_render.py --project templates/template_methods_paper

# 5. Copy final deliverables to repo-level output/
uv run python scripts/pipeline/stage_05_copy.py --project templates/template_methods_paper
```
