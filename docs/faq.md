# Frequently Asked Questions

## Architecture and workflow

### Why can't `src/methods_dsl/` import from `infrastructure/`?

`src/methods_dsl/` is pure model/validation/compilation logic. It must run in
any Python environment with only the standard library installed. All
cross-cutting concerns (rendering, validation, logging) are delegated to
`infrastructure/` and called from `scripts/` — with one declared exception,
`_logging.py`, which is the sanctioned adapter for structured log output
(see `manuscript/layer_contract.yaml`). This keeps the library testable with
zero mocks and reusable outside the pipeline.

### What is the "thin orchestrator" pattern?

`scripts/methods_analysis.py` and `scripts/z_generate_manuscript_variables.py`
should not implement gate, compiler, or scheduling logic. They should:

- Import functions from `src/methods_dsl/`.
- Plot the returned data with matplotlib (the step-count figure).
- Write outputs to `output/` and resolve manuscript tokens.

If you find yourself implementing a validation check or a DAG schedule inside
a script, move it to `src/methods_dsl/` and add a test.

### Why are mocks forbidden?

Mocking tests that a function *was called*, not that it produced the
*correct result*. Pure functions over `Method` dataclasses can be tested with
real fixtures and expected outputs. Mocks give a false sense of correctness
and break the reproducibility guarantee — a mocked `compile_method` could
never actually verify `plan_hash` determinism.

### Why no `.bpl` parser?

Generality a text grammar buys is generality this template exemplar does not
need. Constructing a `Method` directly as frozen Python dataclasses gives
construction-time validation (`__post_init__`) for free and keeps the
exemplar's scope manageable; see `manuscript/07_scope_and_related_work.md`
for the full list of what is deliberately out of scope relative to BPL.

## Testing and coverage

### Why 90% coverage? Can I lower it?

The gate ensures `src/methods_dsl/` logic is thoroughly exercised. If
coverage drops, it signals a missing test — add it, don't lower the gate.

### My new function lowered coverage. What now?

Write tests for the new function in the matching `tests/test_*.py`. Construct
a real `Method`/`Step`/`Quantity` (reuse a `conftest.py` fixture where
possible) with values chosen so the expected gate outcome or plan hash is
exact.

### Do I need to test the analysis script?

Yes. `tests/test_methods_analysis_script.py` runs `run_methods_analysis()`
against a temp output root and asserts real export/report/figure artifacts
are written.

## Manuscript and rendering

### How do I add a new token?

1. Add the key to `generate_variables()` in `src/manuscript_variables.py`,
   sourced from `config.yaml` or an `output/` artifact.
2. Add a test in `tests/test_manuscript_variables.py` asserting the key is
   present.
3. Reference `{{YOUR_TOKEN}}` in a manuscript section.
4. Update [`syntax_guide.md`](syntax_guide.md)'s token table.

### How do I add a new figure?

1. Add the plotting call to `scripts/methods_analysis.py` (matplotlib stays
   in `scripts/`, never in `src/methods_dsl/`) and write the PNG to
   `output/figures/`.
2. In `manuscript/03_results.md`, add a Pandoc image line:
   ```markdown
   ![Caption text.](../output/figures/my_figure.png){#fig:my_label}
   ```
3. Reference it in prose: `See [@fig:my_label].`
4. Update [`syntax_guide.md`](syntax_guide.md) with the new figure label.

### Does this exemplar use `{{VARIABLE}}` token injection?

Yes. Every numeric or run-dependent value in `manuscript/*.md` is a
`{{TOKEN}}` generated by `src/manuscript_variables.py::generate_variables`
and injected by `scripts/z_generate_manuscript_variables.py`. Static
code-derived facts that are not generated per-run instead go in
`data/claim_ledger.yaml`.

## Common pitfalls

### I imported `infrastructure` somewhere other than `_logging.py` and the boundary check failed

`src/methods_dsl/` must stay infrastructure-free outside the one declared
exception. Move that code to `scripts/`, or add it to
`manuscript/layer_contract.yaml`'s `allow_infrastructure_imports` list only
if it is a genuinely sanctioned cross-cutting adapter.

### My test uses `unittest.mock` and the drift gate failed

Replace the mock with a real `Method` (or fixture from `conftest.py`) and
assert the real result. Mocks are forbidden.

### The analysis script runs but `output/figures/` is empty

Check the exit code and look for errors. Ensure you ran the script with
`uv run` from the repository root so imports resolve.

### A `{{TOKEN}}` shows up unresolved in the rendered manuscript

`scripts/z_generate_manuscript_variables.py` prints
`Unresolved {{TOKEN}} in <file>: <name>` for any literal `{{...}}` pattern
in prose that `generate_variables()` does not emit. This includes accidental
generic mentions of the word `TOKEN` inside double braces — never write the
literal placeholder name as illustrative prose; describe the mechanism
without double curly braces instead.
