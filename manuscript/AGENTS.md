# Manuscript (`template_methods_paper/manuscript/`)

Repository-wide agent rules for this exemplar live in
[`../docs/agent_instructions.md`](../docs/agent_instructions.md). This file
covers **manuscript-specific** editing: file roles, the `{{TOKEN}}` /
figure protocol, and the section modification workflow.

This exemplar's manuscript describes a methodology — the controlled-method
specification DSL in `src/methods_dsl/` — rather than results from running
one. It **does** use the `{{VARIABLE}}` token-injection pipeline: every
numeric claim is a `{{TOKEN}}` generated by
`src/manuscript_variables.py::generate_variables` from
`manuscript/config.yaml` and the analysis outputs, injected by
`scripts/z_generate_manuscript_variables.py`. The `config.yaml` block
remains the configuration source of truth for paper metadata.

## Token and Figure Protocol

Numbers are referenced via `{{TOKEN}}` (resolved at render time) and figures
via Pandoc-crossref `[@fig:label]`, never with hardcoded numbers. The one
figure is produced by the thin analysis script from compiled-plan data in
`src/methods_dsl/`:

| Label | PNG Filename | Generator |
|---|---|---|
| `{#fig:step_counts}` | `output/figures/step_counts.png` | `scripts/methods_analysis.py::run_methods_analysis` |

**To add a new token**:

1. Add the key to `generate_variables()` in
   [`../src/manuscript_variables.py`](../src/manuscript_variables.py),
   sourced from `config.yaml` or an `output/` artifact.
2. Add a test in `tests/test_manuscript_variables.py` asserting the key is
   present.
3. Reference `{{YOUR_TOKEN}}` in a manuscript section.

**To add a new figure**:

1. Add the plotting call to
   [`../scripts/methods_analysis.py`](../scripts/methods_analysis.py)
   (matplotlib stays in `scripts/`, never in `src/methods_dsl/`) and write
   the PNG under `output/figures/`.
2. Reference it in a manuscript section with a Pandoc image line and `{#fig:…}`.

## Section Modification Protocol

1. **Update the prose** in `02_methodology.md` / `03_results.md` to match
   the code.
2. **Add or extend tests** in the matching `tests/test_*.py`.
3. **Regenerate analysis outputs and tokens**:
   ```bash
   uv run python projects/templates/template_methods_paper/scripts/methods_analysis.py
   uv run python projects/templates/template_methods_paper/scripts/z_generate_manuscript_variables.py
   ```
4. **Render PDF** from the repository root:
   ```bash
   uv run python scripts/03_render_pdf.py --project templates/template_methods_paper
   ```
5. **Verify the compiled-plan table and figure** appear in the rendered PDF.

## Conventions

1. Avoid boilerplate closers such as "In summary" / "In conclusion" unless
   the section genuinely needs them.
2. Every numeric claim in `03_results.md` must be a `{{TOKEN}}` or a
   `data/claim_ledger.yaml` entry, never a hand-transcribed literal.
3. When referencing template code, prefer concrete paths
   (e.g. `src/methods_dsl/compiler.py::compile_method()`).
